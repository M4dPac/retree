## 🎯 Обзор

**rtree** — современная, быстрая и функциональная консольная утилита для отображения структуры каталогов в виде дерева. Разработана как нативная замена команды GNU `tree` для Windows с полной совместимостью и дополнительными возможностями, адаптированными для Windows-окружения.

### Почему rtree?

- **Нативный для Windows**: Создан специально для Windows с поддержкой функций NTFS — точки соединения, альтернативные потоки данных, атрибуты файлов.
- **Совместимость с GNU tree**: Полноценная замена с поддержкой всех основных флагов GNU tree.
- **Современные возможности**: Иконки Nerd Font, поддержка LS_COLORS, множество форматов вывода.
- **Быстрый**: Написан на Rust для максимальной производительности — эффективно обрабатывает миллионы файлов.
- **Многоязычный**: Полная поддержка русского и английского интерфейса.

## ✨ Возможности

### Основные возможности

- ✅ **Полная совместимость с GNU tree** — поддержка всех основных флагов и опций
- 🎨 **Поддержка LS_COLORS** — цветовое выделение по типам файлов и расширениям
- 🔤 **Иконки Nerd Font** — красивые иконки файлов и папок (требуется Nerd Font)
- 📁 **Множество форматов вывода** — текст, JSON, XML, HTML
- 🔍 **Расширенная фильтрация** — glob-шаблоны, исключения, ограничения глубины
- 📊 **Гибкая сортировка** — по имени, размеру, дате, с поддержкой натуральной сортировки

### Windows-специфичные возможности

- 🪟 **Поддержка NTFS** — точки соединения, символические ссылки, жёсткие ссылки
- 📎 **Альтернативные потоки данных** — просмотр NTFS ADS
- 🔒 **Атрибуты Windows** — скрытые, системные, только для чтения, архивные
- 📏 **Длинные пути** — поддержка путей длиннее 260 символов
- 🌐 **UNC-пути** — поддержка сетевых ресурсов (`\\server\share`)

### Удобство использования

- 🌍 **Многоязычность** — русский и английский интерфейс
- ⚡ **Высокая производительность** — обработка 1М+ файлов без проблем
- 🎯 **Работает без настройки** — готов к использованию из коробки
- 📝 **Настраиваемый** — поддержка файла конфигурации TOML

```powershell
cargo install rtree
```

### Через Scoop

```powershell
scoop bucket add extras

```

### Через Winget

```powershell
winget install rtree
```

### Ручная установка

1. Скачайте последний релиз с [GitHub Releases](https://github.com/user/rtree/releases)
2. Извлеките `rtree.exe` в каталог, указанный в PATH
3. Проверьте установку:

   ```powershell
   rtree --version
   ```

### Сборка из исходников

```powershell
git clone https://github.com/user/rtree.git
cd rtree
cargo build --release
# Бинарный файл будет в target/release/rtree.exe
```

## 🚀 Быстрый старт

```powershell
# Показать текущий каталог
rtree

# Показать конкретный каталог
rtree C:\Users\Имя\Проекты

# Показать скрытые файлы
rtree -a

# Ограничить глубину до 2 уровней
rtree -L 2

# С цветами и иконками
rtree -C --icons

# Только каталоги
rtree -d

# Вывод в JSON
rtree -J > tree.json

# Русский интерфейс
rtree --lang=ru
```

## 📚 Использование

```
rtree [ПАРАМЕТРЫ] [ПУТЬ...]
```

Если путь не указан, используется текущий каталог. Можно указать несколько путей для отображения нескольких деревьев.

### Параметры отображения

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-a` | `--all` | Показывать все файлы, включая скрытые |
| `-d` | `--dirs-only` | Показывать только каталоги |
| `-l` | `--follow` | Следовать по символическим ссылкам |
| `-f` | `--full-path` | Выводить полный путь для каждого файла |
| `-x` | `--one-fs` | Не выходить за пределы текущей файловой системы |
| `-L N` | `--level N` | Ограничить глубину N уровнями |
| | `--filelimit N` | Пропускать каталоги с более чем N элементами |
| | `--noreport` | Не выводить итоговую статистику |

**Примеры:**

```powershell
# Показать все файлы, включая скрытые
rtree -a

# Только каталоги, максимум 3 уровня
rtree -d -L 3

# Пропускать каталоги с более чем 100 файлами
rtree --filelimit 100
```

### Параметры фильтрации

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-P ШАБЛОН` | `--pattern ШАБЛОН` | Показывать только файлы, соответствующие шаблону |
| `-I ШАБЛОН` | `--exclude ШАБЛОН` | Исключить файлы по шаблону (можно повторять) |
| | `--matchdirs` | Применять шаблоны к именам каталогов |
| | `--ignore-case` | Поиск без учёта регистра |
| | `--prune` | Не показывать пустые каталоги |

**Синтаксис glob-шаблонов:**

- `*` — любая последовательность символов
- `?` — любой одиночный символ
- `[abc]` — любой символ из скобок
- `[a-z]` — любой символ из диапазона
- `[!abc]` — любой символ НЕ из скобок

**Примеры:**

```powershell
# Только Rust-файлы
rtree -P "*.rs"

# Исключить node_modules и target
rtree -I "node_modules" -I "target"

# Исходные файлы без учёта регистра
rtree -P "*.[ch]" --ignore-case

# Применить шаблон к каталогам
rtree -P "src*" --matchdirs
```

### Параметры сортировки

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-v` | `--version-sort` | Натуральная сортировка (file2 перед file10) |
| `-t` | `--timesort` | Сортировка по времени модификации |
| `-c` | `--ctime` | Сортировка по времени изменения метаданных |
| `-U` | `--unsorted` | Без сортировки (как выдаёт ОС) |
| `-r` | `--reverse` | Обратный порядок сортировки |
| | `--dirsfirst` | Каталоги перед файлами |
| | `--filesfirst` | Файлы перед каталогами |
| | `--sort ТИП` | Сортировка по: `name`, `size`, `mtime`, `ctime`, `version`, `none` |

**Примеры:**

```powershell
# Натуральная сортировка версий
rtree -v

# По времени модификации, сначала новые
rtree -t -r

# Сначала каталоги, затем файлы по имени
rtree --dirsfirst

# Сортировка по размеру
rtree --sort=size
```

### Параметры вывода

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-i` | `--noindent` | Без отступов и линий |
| `-A` | `--ansi` | Использовать ANSI-графику (├── └──) |
| `-S` | `--cp437` | Использовать CP437-графику |
| `-n` | `--nocolor` | Отключить цвета |
| `-C` | `--color-always` | Всегда использовать цвета |
| | `--color КОГДА` | Режим цвета: `auto`, `always`, `never` |

**Примеры:**

```powershell
# Принудительно включить цвета при перенаправлении
rtree -C | less -R

# Простой ASCII-вывод
rtree -S -n

# Без линий дерева, только отступы
rtree -i
```

### Информация о файлах

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-s` | `--size` | Показывать размер в байтах |
| `-h` | `--human` | Человекочитаемые размеры (1K, 2M, 3G) |
| | `--si` | Единицы СИ (степени 1000 вместо 1024) |
| `-D` | `--date` | Показывать дату модификации |
| | `--timefmt FMT` | Формат даты (синтаксис strftime) |
| `-p` | `--perm` | Показывать права доступа/атрибуты |
| `-u` | `--uid` | Показывать имя владельца (запланировано) |
| `-g` | `--gid` | Показывать группу (запланировано) |
| | `--inodes` | Показывать номера inode |
| | `--device` | Показывать номера устройств |
| `-F` | `--classify` | Добавлять / для каталогов, @ для ссылок, * для исполняемых |
| `-q` | `--safe` | Заменять непечатаемые символы на ? |
| `-N` | `--literal` | Выводить непечатаемые символы как есть |

**Примеры:**

```powershell
# Размеры в человекочитаемом формате
rtree -h

# Даты модификации
rtree -D

# Свой формат даты
rtree -D --timefmt="%Y-%m-%d"

# Права доступа
rtree -p

# Полная информация
rtree -h -D -p -F
```

### Параметры экспорта

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| `-o ФАЙЛ` | `--output ФАЙЛ` | Вывод в файл вместо консоли |
| `-H URL` | `--html URL` | Генерация HTML с базовым URL для ссылок |
| `-T ЗАГОЛОВОК` | `--title ЗАГОЛОВОК` | Заголовок HTML-страницы |
| | `--nolinks` | Не создавать гиперссылки в HTML |
| | `--hintro ФАЙЛ` | Свой HTML-заголовок из файла |
| | `--houtro ФАЙЛ` | Своё HTML-окончание из файла |
| `-X` | `--xml` | Вывод в формате XML |
| `-J` | `--json` | Вывод в формате JSON |

**Примеры:**

```powershell
# Сохранить в файл
rtree -o tree.txt

# Генерация HTML
rtree -H "https://example.com/files" -T "Мой проект" > tree.html

# Экспорт в JSON для обработки
rtree -J | jq '.[] | select(.type == "file")'

# Экспорт в XML
rtree -X > tree.xml
```

### Иконки

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| | `--icons КОГДА` | Показывать иконки: `auto`, `always`, `never` |
| | `--no-icons` | Отключить иконки |
| | `--icon-style СТИЛЬ` | Стиль иконок: `nerd`, `unicode`, `ascii` |

**Примеры:**

```powershell
# Включить иконки Nerd Font
rtree --icons

# Всегда показывать иконки
rtree --icons=always

# Unicode-эмодзи (без Nerd Font)
rtree --icons --icon-style=unicode

# ASCII-иконки
rtree --icons --icon-style=ascii
```

### Windows-специфичные параметры

| Флаг | Полная форма | Описание |
|------|--------------|----------|
| | `--show-streams` | Показывать альтернативные потоки данных NTFS |
| | `--show-junctions` | Показывать точки соединения с целями |
| | `--hide-system` | Скрывать системные файлы даже при `-a` |
| | `--permissions РЕЖИМ` | Формат прав: `posix` или `windows` |
| | `--long-paths` | Принудительно использовать префикс \\?\ |
| | `--lang ЯЗЫК` | Язык интерфейса: `en` или `ru` |

**Примеры:**

```powershell
# Показать альтернативные потоки данных
rtree --show-streams

# Скрыть системные файлы
rtree -a --hide-system

# POSIX-формат прав доступа
rtree -p --permissions=posix

# Русский интерфейс
rtree --lang=ru
```

## 🎨 Настройка цветов

rtree поддерживает формат GNU LS_COLORS для настройки цветов.

### Установка цветов

```powershell
# Через переменную окружения
$env:LS_COLORS = "di=1;34:*.rs=0;33:*.md=0;36"

# Или через TREE_COLORS (имеет приоритет)
$env:TREE_COLORS = "di=1;34:ex=1;32:*.rs=1;33"
```

### Формат цветов

```
ТИП=СТИЛЬ;ПЕРЕДНИЙ_ПЛАН;ЗАДНИЙ_ПЛАН
```

**Типы:**

| Код | Тип |
|-----|-----|
| `di` | Каталог |
| `fi` | Обычный файл |
| `ln` | Символическая ссылка |
| `or` | Битая ссылка |
| `ex` | Исполняемый файл |
| `*.ext` | Файлы с расширением |

**Windows-специфичные типы:**

| Код | Тип |
|-----|-----|
| `hi` | Скрытый файл |
| `sy` | Системный файл |
| `ro` | Только для чтения |
| `jn` | Точка соединения |

**Стили:**

| Код | Стиль |
|-----|-------|
| `0` | Сброс/Обычный |
| `1` | Жирный |
| `2` | Тусклый |
| `3` | Курсив |
| `4` | Подчёркнутый |

**Цвета переднего плана:**

| Код | Цвет | Яркий |
|-----|------|-------|
| `30` | Чёрный | `90` |
| `31` | Красный | `91` |
| `32` | Зелёный | `92` |
| `33` | Жёлтый | `93` |
| `34` | Синий | `94` |
| `35` | Пурпурный | `95` |
| `36` | Голубой | `96` |
| `37` | Белый | `97` |

**256-цветный режим:**

```
38;5;N  # Передний план (N = 0-255)
48;5;N  # Задний план (N = 0-255)
```

**True Color (24-бит):**

```
38;2;R;G;B  # Передний план
48;2;R;G;B  # Задний план
```

### Цветовая схема по умолчанию

```
di=1;34          # Каталоги: жирный синий
ln=1;36          # Ссылки: жирный голубой
or=1;31;40       # Битые ссылки: жирный красный на чёрном
ex=1;32          # Исполняемые: жирный зелёный
*.rs=0;33        # Rust-файлы: жёлтый
*.py=0;33        # Python-файлы: жёлтый
*.js=0;33        # JavaScript: жёлтый
*.md=0;36        # Markdown: голубой
*.txt=0          # Текст: по умолчанию
*.zip=1;33       # Архивы: жирный жёлтый
*.png=1;35       # Изображения: жирный пурпурный
*.jpg=1;35
hi=2;37          # Скрытые: тусклый белый
sy=2;37          # Системные: тусклый белый
```

## 🔨 Сборка из исходников

### Требования

- Rust 1.75 или новее
- Windows 10 1607+ или Windows 11
- (Опционально) Nerd Font для поддержки иконок

### Шаги сборки

```powershell
# Клонировать репозиторий
git clone https://github.com/user/rtree.git
cd rtree

# Собрать в release-режиме
cargo build --release

# Бинарный файл будет в target/release/rtree.exe

# Установить локально
cargo install --path .
```

### Параметры сборки

```powershell
# Сборка со всеми возможностями
cargo build --release --all-features

# Минимальная сборка
cargo build --release --no-default-features

# Кросс-компиляция для другой архитектуры
cargo build --release --target x86_64-pc-windows-msvc
```

## ⚡ Производительность

rtree оптимизирован для эффективной обработки больших структур каталогов.

### Методы оптимизации

- **Итеративный обход** — нет переполнения стека на глубоких каталогах
- **Потоковый вывод** — экономия памяти для больших деревьев
- **Ленивые метаданные** — запрашиваются только при необходимости
- **Скомпилированные шаблоны** — быстрый glob-матчинг
- **Кэширование иконок** — O(1) поиск иконок

## 🔧 Устранение неполадок

### Иконки отображаются некорректно

1. Убедитесь, что установлен Nerd Font (например, «FiraCode Nerd Font»)
2. Настройте терминал на использование Nerd Font
3. Для Windows Terminal добавьте в settings.json:

   ```json
   "profiles": {
     "defaults": {
       "font": { "face": "FiraCode Nerd Font" }
     }
   }
   ```

### Цвета не работают

1. Убедитесь, что используете Windows 10 1607 или новее
2. Включите Virtual Terminal Processing:

   ```powershell
   # PowerShell
   $env:TERM = "xterm-256color"
   ```

3. Попробуйте принудительно включить цвета: `rtree -C`

### Ошибки доступа

- Некоторые каталоги (например, `System Volume Information`) требуют прав администратора
- Используйте `--hide-system` для пропуска системных каталогов
- Запустите от имени администратора для полного доступа

### Длинные пути не работают

1. Включите поддержку длинных путей в Windows:

   ```powershell
   # Запустите от имени администратора
   Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" -Name "LongPathsEnabled" -Value 1
   ```

2. Используйте флаг `--long-paths`

### Проблемы с кодировкой

- rtree по умолчанию использует UTF-8
- Для устаревших приложений используйте `--charset=cp1251` или `--charset=cp866`
- Установите консоль в UTF-8: `chcp 65001`
