# ⚡ Производительность

rtree оптимизирован для эффективной обработки больших деревьев каталогов.

---

## Параллельный режим

```bash
# Авто-определение потоков (по числу ядер ЦП)
rtree --parallel

# Явное указание количества потоков
rtree --parallel --threads 4

# Настройка ёмкости внутренней очереди
rtree --parallel --queue-cap 8192
```

Параллельный режим использует [Rayon](https://github.com/rayon-rs/rayon) с алгоритмом work-stealing для балансировки нагрузки.

> Параллельный режим эффективен начиная с ~10 000 файлов. На малых деревьях накладные расходы на потоки могут незначительно увеличить время выполнения.

---

## Бенчмарки

**Запуск всех бенчмарков:**

```bash
cargo bench
```

**Сохранение baseline:**

```bash
cargo bench -- --save-baseline main
```

**Сравнение с baseline:**

```bash
cargo bench -- --baseline main
```

> Измерение: median time (Criterion) · Режим: `release`

## HTML-отчёты Criterion

После запуска `cargo bench` откройте в браузере:

```
target/criterion/report/index.html
```

Отчёт содержит: медиану, стандартное отклонение, графики распределения и регрессии.
